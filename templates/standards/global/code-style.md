# Code Style Standards

## Naming Conventions

- Use consistent naming with existing codebase
- Follow language conventions (camelCase for JS, snake_case for Python, etc.)
- Descriptive names over abbreviations

## Formatting

- Follow existing indentation patterns in the file
- Keep line lengths reasonable (80-120 chars)
- Use consistent quote style per file

## Imports

- Group imports logically (stdlib, external, internal)
- Remove unused imports
- Avoid circular imports
